Kotlin 클래스와 함수 학습 정리

학습 내용

이번 학습에서는 Kotlin의 클래스 상속, 추상 클래스, 인터페이스, 클래스 위임, 람다 함수에 대해 실습하였다.

클래스 상속과 오버라이딩

Kotlin에서 클래스는 기본적으로 상속이 불가능하므로, 상속을 허용하려면 부모 클래스에 open 키워드를 사용해야 한다.
부모 클래스의 메서드를 자식 클래스에서 재정의하려면 부모 메서드에도 open을 붙이고, 자식 클래스에서는 override를 사용한다.

Shape 클래스를 부모 클래스로 만들고, Circle, Rect, RoundCircle 등의 자식 클래스를 만들어 다형성을 실습하였다.
부모 타입인 Shape 리스트에 여러 자식 객체를 저장하고, 각 객체의 draw() 메서드가 실제 객체 타입에 맞게 실행되는 것을 확인하였다.

또한 is를 사용해 객체 타입을 검사하고, as를 사용해 다운캐스팅하는 방법을 학습하였다.

추상 클래스

추상 클래스는 직접 객체를 생성할 수 없는 클래스이다.
공통적인 구조를 정의하고, 자식 클래스가 반드시 구현해야 하는 메서드를 만들 때 사용한다.

abstract class Animal2 안에 abstract fun bark()를 선언하고, Cat 클래스에서 이를 override하여 구현하였다.
또한 일반 메서드인 run()을 open으로 선언하여 자식 클래스에서 재정의할 수 있도록 하였다.

메서드 오버라이딩과 오버로딩

오버라이딩은 부모 클래스의 메서드를 자식 클래스에서 재정의하는 것이다.
예를 들어 Animal2의 run()을 Cat 클래스에서 다시 구현하였다.

오버로딩은 같은 이름의 메서드를 매개변수의 개수나 타입을 다르게 하여 여러 개 정의하는 것이다.
Cat 클래스에서는 run()과 run(name: String)을 함께 정의하여 오버로딩을 실습하였다.

인터페이스

인터페이스는 클래스가 구현해야 할 기능의 규칙을 정의한다.
인터페이스 안에는 추상 메서드뿐만 아니라 기본 구현이 있는 메서드도 작성할 수 있다.

Flyable 인터페이스를 만들고 Duck 클래스에서 이를 구현하였다.
fly()는 반드시 구현해야 하는 추상 메서드이고, land()는 기본 구현을 가진 메서드이다.

클래스 위임

Kotlin의 클래스 위임은 by 키워드를 사용해 특정 기능의 구현을 다른 객체에게 맡기는 방식이다.

Printer 인터페이스와 ColorPrinter 클래스를 만들고, Logger2 클래스에서 Printer by printer 형태로 위임을 사용하였다.
이를 통해 Logger2가 직접 print()를 구현하지 않아도 전달받은 Printer 객체의 기능을 사용할 수 있다.

람다 함수

람다 함수는 함수를 값처럼 저장하거나 전달할 수 있는 기능이다.
val double: (Int) -> Int = { it * 2 } 형태로 정수를 받아 두 배로 반환하는 람다를 작성하였다.

it은 람다의 매개변수가 하나일 때 사용할 수 있는 기본 이름이다.

고차 함수

고차 함수는 함수를 매개변수로 받거나, 함수를 반환하는 함수이다.

multiplier(factor: Int): (Int) -> Int 함수는 정수를 받아 또 다른 함수를 반환한다.
예를 들어 multiplier(3)(5)는 먼저 factor가 3인 함수를 만들고, 그 함수에 5를 전달하여 결과로 15를 반환한다.

또한 testFun() 함수에서는 (Int) -> Boolean 타입의 함수를 매개변수로 받고, 최종적으로 () -> String 타입의 함수를 반환하는 구조를 학습하였다.

정리

이번 학습을 통해 Kotlin의 객체지향 문법과 함수형 문법을 함께 익혔다.
상속, 오버라이딩, 추상 클래스, 인터페이스, 위임은 객체지향 설계에서 중요한 개념이며, 람다와 고차 함수는 Kotlin에서 함수를 더 유연하게 다루기 위한 핵심 문법이다.
