# Spotify 스타일 백그라운드 음악 플레이어 앱 기획 정리

## 1. 프로젝트 주제

**Spotify 스타일 백그라운드 음악 플레이어 앱**

본 프로젝트는 Spotify와 같은 음악 스트리밍 앱의 사용자 경험을 참고하여, 음악 목록 조회, 검색, 재생 제어, 백그라운드 재생, 알림창 제어, 홈 화면 위젯 제어 기능을 제공하는 Android 음악 플레이어 앱을 구현하는 것을 목표로 한다.

단, 실제 Spotify 음원을 직접 재생하거나 Spotify 계정을 연동하는 방식은 사용하지 않는다. Spotify API, 저작권, DRM, Premium 계정 조건 등의 문제를 피하기 위해 직접 준비한 샘플 음원 또는 로컬 음악 파일을 재생하는 방식으로 구현한다.

---

## 2. 프로젝트 목표

이 앱의 핵심 목표는 사용자가 앱에서 음악을 검색하고 재생할 수 있으며, 앱을 닫아도 음악이 계속 재생되도록 하는 것이다. 또한 알림창과 홈 화면 위젯을 통해 앱을 직접 실행하지 않고도 재생 상태를 제어할 수 있도록 한다.

### 핵심 목표

1. Spotify 스타일의 음악 플레이어 UI 구현
2. 음악 목록 표시
3. 노래 제목 또는 가수명 기반 검색 기능 구현
4. 음악 재생, 일시정지, 다음 곡, 이전 곡 제어
5. 앱을 닫아도 음악이 계속 재생되는 백그라운드 재생 구현
6. 알림창에서 음악 제어
7. 홈 화면 위젯에서 음악 제어
8. 즐겨찾기 및 최근 재생 기록 기능 확장

### 구현하지 않는 것

본 프로젝트에서는 다음 기능은 구현하지 않는다.

1. Spotify 실제 음원 재생
2. Spotify 계정 로그인
3. Spotify API를 이용한 실제 곡 스트리밍
4. 음악 다운로드 기능
5. 유료 음원 사용
6. 실제 결제 또는 구독 기능

### 구현하는 것

본 프로젝트에서는 다음 기능을 구현한다.

1. 샘플 MP3 또는 로컬 음악 파일 재생
2. Spotify 스타일 UI 모방
3. 음악 목록 화면
4. 음악 검색 화면
5. 음악 재생 화면
6. 백그라운드 재생 구조
7. 알림창 음악 제어
8. 홈 화면 위젯 음악 제어
9. 즐겨찾기 및 최근 재생 기록

## 5. 핵심 문제와 해결 방법

### 문제 상황

일반적으로 음악 재생 로직을 Activity 내부에 직접 작성하면 사용자가 앱을 닫거나 화면을 전환했을 때 음악 재생이 중단될 수 있다.

```text
잘못된 구조:

Activity에서 직접 음악 재생
→ 앱 닫힘
→ Activity 종료
→ 음악 종료
```

### 해결 방법

음악 재생 로직을 Activity가 아니라 별도의 Service로 분리한다.

```text
개선된 구조:

Activity는 화면과 버튼 입력만 담당
→ MusicService에 재생 명령 전달
→ MusicService가 실제 음악 재생 담당
→ 앱을 닫아도 Service는 계속 실행
→ 음악 계속 재생
```

이 구조를 사용하면 앱 화면이 종료되어도 음악이 계속 재생될 수 있다.

홈 화면은 사용자가 앱을 실행했을 때 처음 보는 화면이다.

5. 현재 재생 중인 음악 미니 플레이어

## 6.2 검색 화면

검색 화면에서는 사용자가 노래 제목이나 가수명을 입력하여 원하는 음악을 찾을 수 있다.

### 주요 기능

1. 노래 제목 검색
2. 가수명 검색
3. 검색 결과 목록 표시
4. 검색 결과에서 바로 재생

```text
[검색 화면]

검색어 입력: drive

검색 결과
1. Night Drive - Sample Artist
2. Drive Home - City Band
3. Dream Drive - Lo-Fi Studio
```

### 검색 흐름

```text
사용자가 검색어 입력
→ 음악 제목 또는 가수명 기준으로 필터링
→ 검색 결과 표시
→ 노래 선택
→ MusicService에 재생 명령 전달
→ 백그라운드 재생 시작
```

## 7. 홈 화면 위젯 기능

홈 화면 위젯은 사용자가 앱을 직접 실행하지 않고도 음악을 제어할 수 있게 해주는 기능이다.

### 위젯 주요 기능

1. 현재 재생 중인 곡 제목 표시
2. 가수명 표시
3. 이전 곡 버튼
4. 재생 / 일시정지 버튼
5. 다음 곡 버튼
6. 검색 화면 열기 버튼

### 위젯 UI 예시

```text
┌─────────────────────────┐
│  현재 재생 중             │
│  Night Drive             │
│  Sample Artist           │
│                         │
│  ⏮    ⏯    ⏭            │
│                         │
│  🔍 노래 검색             │
└─────────────────────────┘
```

### 위젯 동작 흐름

```text
홈 화면 위젯에서 다음 곡 버튼 클릭
→ PendingIntent 실행
→ BroadcastReceiver가 NEXT 명령 수신
→ MusicService에 다음 곡 명령 전달
→ ExoPlayer가 다음 곡으로 변경
→ 위젯의 현재 곡 정보 갱신
```

## 9. 데이터 구조 예시

음악 데이터를 표현하기 위한 기본 구조는 다음과 같이 설계할 수 있다.

```kotlin
data class Music(
    val id: Long,
    val title: String,
    val artist: String,
    val albumImage: String,
    val fileName: String,
    val duration: Long,
    val isFavorite: Boolean = false
)
```

### 필드 설명

| 필드명 | 설명 |
|---|---|
| id | 음악 고유 번호 |
| title | 노래 제목 |
| artist | 가수명 |
| albumImage | 앨범 이미지 경로 또는 URL |
| fileName | 음악 파일 이름 또는 경로 |
| duration | 음악 길이 |
| isFavorite | 즐겨찾기 여부 |

## 10. 검색 기능 예시

처음 구현 단계에서는 복잡한 검색 알고리즘 대신 제목과 가수명을 기준으로 단순 필터링하는 방식으로 구현한다.

```kotlin
fun searchMusic(keyword: String, musicList: List<Music>): List<Music> {
    return musicList.filter { music ->
        music.title.contains(keyword, ignoreCase = true) ||
        music.artist.contains(keyword, ignoreCase = true)
    }
}
```

### 검색 기준

1. 노래 제목에 검색어가 포함되어 있는지 확인
2. 가수명에 검색어가 포함되어 있는지 확인
3. 대소문자는 구분하지 않음
4. 조건에 맞는 음악만 결과로 표시

## 16. 발표용 설명 문장

본 프로젝트는 Spotify와 같은 음악 스트리밍 앱의 핵심 사용자 경험을 참고하여, 백그라운드 재생과 홈 화면 제어 기능을 제공하는 음악 플레이어 앱을 구현한다. 사용자는 앱에서 음악을 검색하고 재생할 수 있으며, 앱을 닫은 후에도 음악이 계속 재생된다. 또한 알림창과 홈 화면 위젯을 통해 앱을 직접 실행하지 않고도 재생/일시정지, 다음 곡, 이전 곡 기능을 사용할 수 있다.

기존에 Activity 내부에서 음악을 직접 재생하는 방식은 앱 화면이 종료되면 음악 재생도 함께 중단될 수 있다는 한계가 있다. 본 프로젝트에서는 음악 재생 기능을 별도의 MusicService로 분리하여 이러한 문제를 해결한다. Activity는 UI와 사용자 입력만 담당하고, 실제 음악 재생은 Service에서 처리함으로써 안정적인 백그라운드 재생을 지원한다.

## 17. 최종 정리

최종 프로젝트 방향은 다음과 같다.

```text
Spotify 실제 연동 앱이 아니라,
Spotify 스타일 백그라운드 음악 플레이어 앱으로 만든다.
```

### 핵심 차별점

1. 앱을 닫아도 음악이 계속 재생된다.
2. 홈 화면 위젯에서 바로 제어할 수 있다.
3. 알림창에서 재생 상태를 제어할 수 있다.
4. 검색 기능으로 원하는 노래를 빠르게 찾을 수 있다.
5. Spotify 스타일 UI를 통해 익숙한 음악 앱 경험을 제공한다.

이 방향은 Spotify 실제 연동보다 구현 난이도가 낮고, 모바일 프로그래밍 과제로 보여줄 수 있는 Android 핵심 기능이 많다. 특히 백그라운드 재생, 알림창 제어, 홈 화면 위젯, 검색 기능을 포함하면 단순 CRUD 앱보다 완성도가 높아 보인다.
