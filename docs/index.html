핵심 기능

사용자 관점에서 완성한 화면 흐름

Home

음악을 발견하는 첫 화면

추천 카드와 전체 목록을 배치해 앱을 열자마자 재생할 음악을 고를 수 있습니다.

Search

검색 결과 중심 화면

검색어가 바뀌면 결과 목록이 즉시 갱신되어 원하는 곡으로 빠르게 이동합니다.

Player

상세 재생 화면

앨범 영역, 곡 정보, 재생 시간, 이전/재생/다음 버튼, 큐 미리보기를 제공합니다.

Library

보관함 화면

즐겨찾기와 최근 선택한 음악을 모아 다시 듣기 쉬운 구조로 정리했습니다.

앱 구조

화면, 상태, 재생 로직을 분리했습니다

Compose 화면은 사용자가 보는 UI를 담당하고, ViewModel은 검색과 선택 상태를 관리합니다. 재생 로직은 별도 컨트롤러와 서비스 구조로 분리해 음악 앱에 필요한 확장성을 확보했습니다.

시연 흐름

발표에서 보여줄 5단계

  1. 홈 화면 확인
    음악 목록, 추천 영역, 미니 플레이어 위치를 소개합니다.
  2. 검색 사용
    곡 제목이나 가수명을 입력해 결과가 필터링되는 모습을 보여줍니다.
  3. 곡 선택
    원하는 곡을 선택하고 상세 플레이어 화면으로 이동합니다.
  4. 재생 제어
    재생/일시정지, 이전 곡, 다음 곡, 진행률 조작을 시연합니다.
  5. 보관함 확인
    즐겨찾기와 최근 선택 목록이 사용 흐름에 연결되는 것을 보여줍니다.

확장 방향

음악 앱다운 사용성을 더 넓히는 다음 단계

1

백그라운드 재생 강화

앱을 벗어나도 재생 상태가 안정적으로 유지되도록 서비스 흐름을 더 다듬습니다.

2

알림 제어

알림창에서 현재 곡을 확인하고 재생을 제어할 수 있는 경험으로 확장합니다.

3

위젯과 저장 기능

홈 화면 위젯과 즐겨찾기 저장 기능을 추가해 앱 밖에서도 접근성을 높입니다.

마무리

목록에서 재생까지 이어지는 음악 플레이어 경험

이 프로젝트는 음악 목록을 보여주는 데서 끝나지 않고, 검색, 선택, 재생 화면, 미니 플레이어, 백그라운드 재생을 고려한 구조까지 하나의 흐름으로 연결했습니다.
